Question : 1 -> What is Map,filter,forEach,Find between differance ?
Answer : Map, filter, forEach, and find are all array methods in JavaScript that allow you to perform operations on arrays. The main differences between them are:


1. Map: The map() method creates a new array by calling a function on each element of the original array. The function can perform any transformation on the original element and return a new value. The resulting values are then added to the new array in the same order as the original array. The original array is not modified


Example: const numbers = [1, 2, 3, 4];
const doubled = numbers.map(num => num * 2);


2.Filter: The filter() method creates a new array by calling a function on each element of the original array. The function should return a boolean value (true or false) to indicate whether the element should be included in the resulting array. If the function returns true, the element is included, otherwise it is excluded. The original array is not modified.


Example: const numbers = [1, 2, 3, 4];
const evenNumbers = numbers.filter(num => num % 2 === 0);


3.forEach: The forEach() method calls a function on each element of the original array. The function can perform any operation on the element, but it doesn't return anything. The original array is not modified.


Example: const numbers = [1, 2, 3, 4]; numbers.forEach(num => console.log(num * 2));;


4.Find: The find() method returns the first element in the array that satisfies a given condition. The condition is defined by a function that is called on each element of the array. If the function returns true for an element, that element is returned. Otherwise, undefined is returned.


Example: const numbers = [1, 2, 3, 4]; const found = numbers.find(num => num > 2);

Question : 2 -> What is LocalStorage, Session storage, cookies between differencee ?
Answer : LocalStorage and Session Storage are storage mechanisms in web browsers that allow web pages to store data locally on a user's device,

while cookies are small text files that are stored on a user's device by a website.

LocalStorage has a larger storage limit than cookies and persists even after the browser is closed, while Session Storage is only available during the current session. Cookies have a smaller storage limit than LocalStorage, but they can be set to expire after a specified time.
Question : 3 -> What is what is Event Loop ?
Answer : The Event Loop is a mechanism in JavaScript that allows it to handle multiple tasks simultaneously without blocking the execution of other code. It is the core of JavaScript's concurrency model and is responsible for handling all asynchronous code in JavaScript.

In JavaScript, all code runs in a single thread, which means that only one task can be executed at a time. This can be a problem for applications that need to handle multiple tasks simultaneously, such as responding to user input while downloading data from a server. The Event Loop helps solve this problem by allowing asynchronous tasks to be executed without blocking the main thread.
